API 使用指南

会话类接口需要 LinuxDo 登录且带 token_atlas_session Cookie。API Key 接口使用 X-API-Key。

GET /me

返回当前用户、额度与领取统计。

POST /me/claim

使用会话 Cookie 领取账号，例如 {"count": 2}。

curl -X POST ${TOKEN_PROVIDER_BASE_URL}/me/claim \
\
  -H "Content-Type: application/json" \
  --cookie "token_atlas_session=YOUR_SESSION" \
  -d "{\"count\": 2}"
GET /me/claims

列出当前会话已领取的 JSON 账号。

GET /me/claims/archive

下载已领取 JSON 账号的 ZIP 打包。

POST /api/claim

使用 X-API-Key 领取账号，例如 {"count": 2}。

curl -X POST ${TOKEN_PROVIDER_BASE_URL}/api/claim \
\
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d "{\"count\": 2}"
GET /api/download/{token_id}

通过领取结果里的 token_id 下载 JSON。

curl -L ${TOKEN_PROVIDER_BASE_URL}/api/download/123 \
  -H "X-API-Key: your-api-key"
GET /json

索引接口（X-API-Key 认证）。

GET /json/item

可用 name、id 或 index 获取详情，例如 /json/item?name=demo.json。

POST /json/archive

按名称打包下载 ZIP，Body: {"names": ["a.json", "b.json"]}。

GET /zip

下载全部 JSON 文件 ZIP。
