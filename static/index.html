API 接口体系与开发者服务指引

此面板提供了以程序化方式访问 Token Atlas 数据的所需端点。支持外部调度轮询、批量聚合与系统级心跳拨测功能，帮助开发者快速完成后端资源的横向集成。

系统采用统一安全密钥鉴权。您需要在受到保护的请求头中附带 X-Access-Key: [你的密码]，或者将密钥通过 query parameter 附加: ?key=[你的密码]。
POST /auth/login

创建登录会话授权 (Session Auth)，进行密码有效性检测。

curl -X POST http://127.0.0.1:8000/auth/login -H "Content-Type: application/json" -d "{\"password\":\"你的密码\"}"
curl -X POST "http://127.0.0.1:8000/auth/login?key=你的密码"
GET /json Auth Required

获取完整的可用索引树，包含文件大小、时间及元数据。

curl http://127.0.0.1:8000/json -H "X-Access-Key: 你的密码"
GET /json/item Auth Required

通过传入特定的过滤条件来精确回拨详情与完整的 Payload 内容。
可选参数: name (文件名), id (UUID), index (数字序号)

curl "http://127.0.0.1:8000/json/item?index=0&key=你的密码"
curl "http://127.0.0.1:8000/json/item?name=sample.json&key=你的密码"
GET /zip Auth Required

将远端 token 目录下的全部 JSON 数据流式压缩为单个 ZIP 归档包。

curl -O -J "http://127.0.0.1:8000/zip?key=你的密码"
GET /health

获取系统整体运行状况拨测，免密码可用。

curl http://127.0.0.1:8000/health
GET /?key={password}

自动免密码跳转直接访问 Web 控制台 UI 的快捷链接方式。
