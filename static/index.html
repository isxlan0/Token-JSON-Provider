API Docs

Session APIs require LinuxDo login and a valid token_atlas_session cookie. API Key APIs use X-API-Key.

GET /me

Return current user, quota, and claim stats.

POST /me/claim

Claim accounts with session cookie, e.g. {"count": 2}.

curl -X POST http://127.0.0.1:8000/me/claim \
  -H "Content-Type: application/json" \
  --cookie "token_atlas_session=YOUR_SESSION" \
  -d "{\"count\": 2}"
GET /me/claims

List claimed JSON accounts for the current session.

GET /me/claims/archive

Download a ZIP archive of claimed JSON accounts.

POST /api/claim

Claim accounts with X-API-Key header, e.g. {"count": 2}.

curl -X POST http://127.0.0.1:8000/api/claim \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d "{\"count\": 2}"
GET /api/download/{token_id}

Download JSON by token_id from claim response.

curl -L http://127.0.0.1:8000/api/download/123 \
  -H "X-API-Key: your-api-key"
GET /json

Index API with X-API-Key header.

GET /json/item

Fetch by name, id, or index. Example: /json/item?name=demo.json.

POST /json/archive

Download ZIP for selected names. Body: {"names": ["a.json", "b.json"]}.

GET /zip

Download ZIP for all JSON files.
