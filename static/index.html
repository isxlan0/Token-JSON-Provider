接口文档

受保护接口支持两种鉴权方式，任选其一：

X-Access-Key: 你的访问密码
?key=你的访问密码

1. 登录校验

POST /auth/login
Content-Type: application/json

{"password":"你的访问密码"}
POST /auth/login?key=你的访问密码

2. 获取全部索引

GET /json
GET /json?key=你的访问密码

3. 通过文件名获取详情

GET /json/item?name=文件名.json
GET /json/item?name=文件名.json&key=你的访问密码

4. 打包下载当前结果

POST /json/archive
Content-Type: application/json

{"names":["a.json","b.json"]}
不传 names 或传空数组时，返回全部 token 文件的 ZIP。

5. 健康检查

GET /health

调用示例

curl -X POST http://127.0.0.1:8000/auth/login -H "Content-Type: application/json" -d "{\"password\":\"your-access-key\"}"
curl http://127.0.0.1:8000/json -H "X-Access-Key: your-access-key"
curl -X POST http://127.0.0.1:8000/json/archive -H "X-Access-Key: your-access-key" -H "Content-Type: application/json" -d "{\"names\":[\"demo.json\"]}" --output token.zip
